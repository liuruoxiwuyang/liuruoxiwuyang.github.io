强缓存和协商缓存

发表于 2020-01-21 | 分类于 HTTP

浏览器在第一次请求发生后，再次请求时：
1、如果命中强缓存（也叫本地缓存）则直接从缓存中获取资源信息，不会与服务器发生通信；
2、如果没有命中强缓存，则将请求发送到服务器，由服务器判断是否命中协商缓存，如命中（状态码304），则从缓存中获取资源（但服务器会返回新的响应header信息更新缓存中的对应header信息）。否则从服务器获取最新的资源内容。
